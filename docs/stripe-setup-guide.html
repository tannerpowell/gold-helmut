Gold Helmet Award Stripe Setup Guide

Setting Up Stripe for Donations

This guide walks you through creating a Stripe account, grabbing your API keys, and testing a donation. Takes about 15 minutes. No coding required.

1 Create Account 2 Get Test Keys 3 Share Keys 4 Test a Donation 5 Webhook (optional) 6 Go Live
1

Create a Stripe Account

  1. Go to dashboard.stripe.com/register
  2. Enter your email, full name, and a password. Click Create account.
  3. Stripe will send a verification email. Click the link to confirm.
  4. You'll land on the Stripe Dashboard. You're in test mode by default (an orange "Test mode" banner at the top). That's exactly what we want for now.
🛈

You don't need to complete business verification to test donations. Stripe lets you use test mode immediately.

2

Get Your Test API Keys

  1. In the Stripe Dashboard, click Developers in the top nav bar.
  2. Click API keys in the left sidebar.
  3. Make sure the toggle at the top says "Test mode" (it should be orange).
  4. You'll see two keys:
Your two keys
Publishable key pk_test_51ABC...
Secret key sk_test_51ABC...

The publishable key is visible on-screen. For the secret key, click "Reveal test key" to see it. Copy both.

⚠

Never share your secret key publicly. It grants full access to your Stripe account. Send it only through secure channels (password manager share, encrypted message, etc).

3

Share Your Keys With the Developer

Send the following 3 values to your developer. They'll drop them into the site's configuration file. Use a secure method (1Password share link, Signal message, etc).

Values to send
Publishable key pk_test_51ABC...your full key
Secret key sk_test_51ABC...your full key
Site URL https://yourdomain.com

Your developer will place these into a file called .env.local, which looks like this: 

# .env.local
NEXT_PUBLIC_STRIPE_PUBLIC_KEY=pk_test_51ABC...
STRIPE_SECRET_KEY=sk_test_51ABC...
NEXT_PUBLIC_SITE_URL=https://yourdomain.com
🛈

For testing, the site URL stays http://localhost:3000. Your developer only needs the two Stripe keys to test locally.

4

Test a Donation

Once your developer has the keys installed, they'll send you a link to the test site. Here's how to run through a test donation:

  1. Open the site and scroll to the Fund a Scholarship section.
  2. Pick a donation amount (or type a custom one) and click the Donate button.
  3. You'll be redirected to Stripe's checkout page. Use this test credit card:
Test card details
Card number 4242 4242 4242 4242
Expiry Any future date (e.g. 12/30)
CVC Any 3 digits (e.g. 123)
ZIP Any 5 digits (e.g. 80202)
  1. Click "Donate" on the Stripe page. You'll be redirected back to a Thank You page showing the amount.
  2. Go to your Stripe Dashboard → Payments and you should see the test payment listed.
✔

No real money is charged in test mode. The 4242 card is Stripe's magic test number. You can run as many test donations as you want.

Want to test a declined card? Use 4000 0000 0000 0002 instead. It will show a decline error.

5

Set Up the Webhook (optional)

Webhooks let the site get notified when a donation completes, even if the donor closes their browser before the "Thank You" page loads. This is optional for launch but recommended.

  1. In the Stripe Dashboard, go to Developers → Webhooks.
  2. Click "Add endpoint".
  3. Enter your endpoint URL: 
https://yourdomain.com/api/webhook
  1. Under "Select events to listen to", search for and select:
    checkout.session.completed
  2. Click "Add endpoint" to save.
  3. On the endpoint detail page, click "Reveal" under Signing secret. Copy the value (starts with whsec_).
  4. Send this signing secret to your developer. They'll add it as: 
STRIPE_WEBHOOK_SECRET=whsec_abc123...
🛈

For local testing, your developer will use the Stripe CLI (stripe listen) which generates its own webhook secret. You only need to set this up for the live/staging site.

6

Go Live

When you're ready to accept real donations, you need to do two things: activate your Stripe account and swap in live keys.

A. Activate your account

  1. In the Stripe Dashboard, click "Activate your account" (or the banner at the top).
  2. Fill in your business details:
    • Business type: Non-profit / 501(c)(3)
    • Business name: Gold Helmet Award Corp
    • EIN: Your organization's EIN
  3. Add a bank account for payouts (where donations get deposited).
  4. Complete identity verification for the account representative.
⚠

Stripe typically reviews non-profit applications within 1 to 2 business days. You can continue testing while waiting.

B. Get live keys

  1. Go to Developers → API keys.
  2. Toggle off "Test mode" (the switch at the top). You should see a green "Live" indicator.
  3. Copy your live publishable key (pk_live_...) and live secret key (sk_live_...).
  4. Send both to your developer, the same way you sent the test keys. They'll update the production environment.
🛑

Live keys charge real money. Double-check the amounts on your site before sharing live keys. The test card (4242...) won't work with live keys.

If you set up a webhook in Step 5 with a test endpoint, you'll need to create a new live webhook endpoint the same way (Developers → Webhooks, but with test mode off).

Common Questions

What does Stripe charge per donation?

2.9% + $0.30 per transaction. On a $100 donation, you receive $96.80. Non-profits can apply for discounted rates.

When do donations reach my bank account?

Stripe pays out on a rolling 2-business-day schedule by default. You can change this in Dashboard → Settings → Payouts.

Can I issue refunds?

Yes. Go to Dashboard → Payments, click on the payment, and click "Refund". You can refund partially or in full.

Do donors get a receipt?

Stripe sends an email receipt automatically. You can customize the receipt in Dashboard → Settings → Emails.

I need help with something not covered here.

Stripe's support is excellent: support.stripe.com. For site-specific questions, contact your developer.
